ESTRUCTURA v1.0 - 06/09/2016

      La organización de la estructura consistira en:
      # RESET
        Anula los posibles estilos heredados de otros CSSs o del navegador.

      # DISEÑO
        Incorporación del diseño final.

        # CSSs GLOBALES.
          CSSs de los elementos, clases e IDs a nivel global.

        # CSSs LOCALES.
          CSSs de los elementos, clases e IDs a nivel local por secciones.

          # RESPONSIVE
            Responsive de los valores locales de la sección a la que pertenece.


      PRIORIDADES DE COLOCACIÓN
      El orden de colocación de los CSSs es muy importante.
      Llendo de los CSSs más globales a más locales.
      Esto significa que los CSSs mas adelantados serian los que afecten de una manera general.
      Y los más atrasados serian los que afecten a detalles muy especificos.

      De igual manera se haran los detalles dentro de los bloques de CSS.
      Primero los ELEMENTOS que afecta de una manera más general, seguido de las CLASS y terminando en los IDs
        1º ELEMENTO
        2º CLASS
        3º ID

      EVOLUCIÓN DE ESTILOS EN CASCADA
      Segun la estructura de abajo se puede ver el proceso evolutivo de la CLASS '.show' desde el resetado de
      CSSs heredados que no queremos, hasta los estilos que son condicionados por la estructura.

      CARGA DE CSS
      Los CSSs pueden ser afectados por otros CSS que no podemos controlar.
      Por eso, para que las propiedades no se vean afectadas, se añade al
      path ELEMENTOS o IDs que pueda afectar al CSSs por igual (ejem. html head + body).

      Esto daria más prioridad a nuestro CSS respecto a otros CSSs externos.

      Se tenda que mantener la formula en todo el CSS para que las prioridades no se vean afectadas.


    /*RESET DEL CSS ANTERIOR*/
      html head + body{font-size:inherit;width:auto;...}
      html head + body a{font-size:inherit;width:auto;...}
      html head + body .show{font-size:inherit;width:auto;...} <-- La herencia de CSSs de la class 'show' es reseteada
      html head + body .hide{font-size:inherit;width:auto;...}
      html head + body #head{font-size:inherit;width:auto;...}
      html head + body #document{font-size:inherit;width:auto;...}
      html head + body #resultlist{font-size:inherit;width:auto;...}
    /*RESET DEL CSS ANTERIOR*/

    /*NUEVO DISEÑO CSS*/
      /*VALORES GLOBALES DE LOS ELEMENTOS Y CLASES*/
        html head + body{...}
        html head + body *{...}
        html head + body a{...}
        html head + body a:hover{...}
        html head + body [type="text"]{...}
        html head + body .show{...} <- Se añade a la class 'show' los nuevos valores globales
        html head + body .hide{...}
      /*VALORES GLOBALES DE LOS ELEMENTOS Y CLASES*/

      /*VALORES LOCALES DE LOS ELEMENTOS Y CLASES SEPARADOS POR SECCIONES*/
        /*SECCION DEL HEADER*/
          html head + body #head a{...}
          html head + body #head p{...}
          html head + body #head h1{...}
          html head + body #head .show{...} <- Se añade a la class 'show' los nuevos valores locales ajustados a la sección 'head'
          html head + body #head .hide{...}
          html head + body #head .up{...}
          html head + body #head .down{...}
          html head + body #head .show .up{...}
          html head + body #head .show .down{...}

          /*RESPONSIVE DE LA SECCION ANTERIOR*/

              html head + body #head .hide{...}
              html head + body #head .up{...}
              html head + body #head .down{...}
              html head + body #head .active .show{...} <- Condicional de '.show' en '#head' que solo se activaria en caso de ser hijo de '.active'
              html head + body #head .active ~ .class .show{...} <-Condicional de '.show' en '#head' que solo se activaria en caso de que '.active' se encuentre como ancestro en la herencia
            }
